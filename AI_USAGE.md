### Example Usage

`Generate a ci.yml that checks typescript types & lint for a project uses the following package.json. 
Package.json: ...`

The AI-generated result was:

```yml
name: Node.js CI

# Trigger the action on push or pull request events
on:
  push:
    branches: ["main", "master"]
  pull_request:
    branches: ["main", "master"]

jobs:
  quality-check:
    runs-on: ubuntu-latest

    steps:
      # 1. Check out the code from the repository
      - name: Checkout code
        uses: actions/checkout@v4

      # 2. Install Node.js
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20" # Change this to match your app's version
          cache: "npm" # Caches dependencies to make subsequent runs faster

      # 3. Install dependencies
      - name: Install dependencies
        # 'npm ci' is preferred over 'npm install' for CI environments
        # because it enforces the exact versions in package-lock.json
        run: npm ci

      # 4. Check TypeScript Types
      - name: Run Type Check
        run: npm run type-check

      # 5. Run Linter
      - name: Run Linter
        run: npm run lint
```

This workflow was generated by Claude Sonnet 4.5. As you can see, it has issues:

- It uses outdated versions of `actions/checkout` and `actions/setup-node`.
- The Node.js version does not match my project's version. This project uses Node v24.
- PR is not required in this case-study.
- My default branch is `main`

So, I manually updated the script:

```yml
# Bu Workflow AI ile üretildi.
name: Node.js CI

on:
  # Sadece Push için gerekli olsun, PR için bu case-study'de gerek yok.
  push:
    branches: ["main"]

jobs:
  quality-check:
    runs-on: ubuntu-latest

    steps:
      # 1. Check out the code from the repository
      - name: Checkout code

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "24"
          cache: "npm"

      # 3. Install dependencies
      - name: Install dependencies
        # 'npm ci' is preferred over 'npm install' for CI environments
        # because it enforces the exact versions in package-lock.json
```

If I were doing this manually without AI, I would need to Google syntax and GitHub Actions references every time. AI helps reduce that overhead.

I asked the AI:

> Optimize the style structure of this React Native component using the following rules:
>
> 1.  Do not use inline styles.
> 2.  Do not change the UI.

The improved version was:

```ts
import {
  blueToPurpleGradientLowOpacity,
  Colors,
  Fonts,
} from "@/constants/theme";
import { LinearGradient } from "expo-linear-gradient";
import { Image, StyleSheet, Text, TouchableOpacity, View } from "react-native";

const colors = Colors.dark;

export interface PreviewParameterProps {
  prompt: string;
  style: string;
}

export function PreviewParameters({ prompt, style }: PreviewParameterProps) {
  return (
    <View style={styles.container}>
      <View style={styles.baseBackground} />

      <LinearGradient
        colors={blueToPurpleGradientLowOpacity as any}
        locations={[0.25, 1]}
        start={{ x: 0, y: 0 }}
        end={{ x: 1, y: 0 }}
        style={styles.gradientOverlay}
      />

      <View style={styles.content}>
        <View style={styles.headerRow}>
          <Text style={styles.label}>Prompt</Text>

          <TouchableOpacity activeOpacity={0.8}>
            <View style={styles.copyRow}>
              <Image
                source={require("@/assets/images/copy.png")}
                style={styles.copyIcon}
              />
              <Text style={styles.copyText}>Copy</Text>
            </View>
          </TouchableOpacity>
        </View>

        <Text style={styles.promptText}>{prompt}</Text>

        <View style={styles.styleBadge}>
          <Text style={styles.styleText}>{style}</Text>
        </View>
      </View>
    </View>
  );
}

const RADIUS = 12;

const styles = StyleSheet.create({
  container: {
    padding: 12,
    borderRadius: RADIUS,
    overflow: "hidden",
  },

  baseBackground: {
    ...StyleSheet.absoluteFillObject,
    backgroundColor: colors.backgroundSoft,
    borderRadius: RADIUS,
  },

  gradientOverlay: {
    ...StyleSheet.absoluteFillObject,
    borderRadius: RADIUS,
  },

  content: {
    gap: 12,
  },

  headerRow: {
    flexDirection: "row",
    justifyContent: "space-between",
    alignItems: "center",
  },

  label: {
    color: colors.text,
    fontFamily: Fonts.manropeBold,
    fontSize: 15,
  },

  copyRow: {
    flexDirection: "row",
    alignItems: "center",
    gap: 6,
  },

  copyIcon: {
    width: 16,
    height: 16,
  },

  copyText: {
    color: colors.textDark400,
    fontSize: 11,
  },

  promptText: {
    color: colors.text,
    fontSize: 16,
    fontFamily: Fonts.manropeRegular,
  },

  styleBadge: {
    minWidth: 76,
    maxWidth: 100,
    alignSelf: "flex-start",
    borderRadius: 50,
    backgroundColor: "rgba(250, 250, 250, 0.1)",
    paddingHorizontal: 8,
    paddingVertical: 4,
    alignItems: "center",
    justifyContent: "center",
  },

  styleText: {
    fontSize: 12,
    fontFamily: Fonts.manropeRegular,
    color: colors.text,
  },
});
```

As you can see, it keeps the UI intact while moving all inline styles into a clean, maintainable style sheet. This significantly improves readability.
